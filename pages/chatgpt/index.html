Summary

Design Insights

The Problem

ChatGPT is one of the most widely used generative AI products, but its active workspace still has opportunities for refinement. The current experience can feel more organized around product features than user intent, making it harder for new, casual, and everyday users to understand where to go, what to do, and how to make the workspace their own.

This redesign focuses on four core issues that may impact how users engage with ChatGPT as a product:

  • The current workspace can feel feature-led instead of user-intent-led, requiring users to understand product-specific terms before certain areas feel meaningful.
  • The sidebar shows opportunities for stronger hierarchy, with limited distinction between personal workspace areas, system tools, and product features.
  • Key controls can feel disconnected from the user’s moment of action. For example, starting a chat or generation is one of ChatGPT’s core actions, so it should feel like a primary action rather than another navigation item.
  • Advanced AI options could be explained more clearly, reducing guesswork around which models, GPTs, or tools are best suited for a user’s task.

While other areas could continue to evolve, these four issues became the focus because they directly relate to workspace clarity, navigation, and user confidence.

The Solution

The solution was to redesign ChatGPT’s active workspace around user intent rather than product feature exposure. I restructured the sidebar, prompt area, and model controls so users could more easily understand where to go, what to do, and how to configure ChatGPT at the moment they are ready to engage.

The redesign supports user intent through the following decisions:

  1. I reorganized the sidebar around user intent, separating personal workspace areas, custom GPTs, and system-level tools so the interface feels easier to scan and navigate.
  2. I reduced redundant entry points and gave the sidebar a clearer hierarchy, helping it function more like a workspace hub instead of a list of competing product features.
  3. I moved model configuration closer to the prompt field so users could choose how ChatGPT should respond at the moment they are preparing to type or generate.
  4. I created a compact design system and interaction-state frames to show the redesigned workspace, reusable components, and key behavior patterns without turning the project into a full end-to-end product rebuild.

The Outcome

The redesigned active workspace is intended to better support a user-intent-led experience. By reorganizing the sidebar, reducing redundant entry points, and moving model controls closer to the prompt field, the concept aims to make the interface easier to understand, easier to navigate, and more direct to engage with.

The final design file supports this direction through a compact design system, before-and-after workspace frames, and key interaction states. Together, these artifacts show how the redesign explores improved clarity without needing to rebuild ChatGPT as a full end-to-end product.
