Product Design

ChatGPT — Redesigning Interface Clarity, Conversation, and Navigation Roles

A structural redesign that separates conversation and navigation to reduce cognitive friction and create a calmer, more focused communication experience.

01 — Problem

ChatGPT’s interface combined conversation, navigation, and system controls within the same visual space.

The core experience — conversation — was frequently interrupted by navigation elements, project folders, and structural controls that behaved more like a file system than a communication tool.

This forced users to split attention between interacting and managing the system. Tasks such as switching conversations or locating previous chats introduced unnecessary friction and disrupted conversational flow.

The structure was system-oriented, not user-oriented.

02 — Why It Mattered

ChatGPT is a conversation-first product. The conversation area functions as the primary workspace.

This resulted in:

  • Reduced focus due to competing interface elements
  • Slower interaction flow when switching or managing conversations
  • Increased cognitive load from system-driven structure

For a product centered around thinking and communication, this friction directly impacts usability.

03 — Solution

Redesigned the interface to separate conversation from navigation using a clear, role-based layout.

Key decisions:
  • Preserved the main conversation area as a dedicated, uninterrupted workspace
  • Consolidated all navigation and conversation management into a persistent left panel
  • Established clear functional boundaries between interaction and system controls

Structure:
  • Left panel- navigation and conversation management
  • Center panel — active conversation

This aligns the interface with user mental models rather than system architecture.

04 — Impact (Design Rationale)

The new structure reduces cognitive load by eliminating competing interface responsibilities within the same space.

It improves:

  • Focus by isolating the primary task (conversation)
  • Predictability through consistent spatial organization
  • Learnability by aligning with familiar application patterns

Users no longer need to manage system structure while interacting.

05 — Skills Demonstrated

  • Interaction design for conversational interfaces
  • Information architecture for multi-layered systems
  • Cognitive load reduction in task-based environments
  • Spatial organization and layout strategy
  • Designing around user mental models vs system structure

06 — Outcome

The redesigned interface improves usability by clarifying roles and reducing friction.

Users can:

  • Navigate conversations without interrupting flow
  • Maintain focus on active interaction
  • Quickly understand where actions and controls are located

The system becomes easier to use immediately and provides a scalable structure for future features.

Insight

"Design the interface around how users think, not how systems store information."
