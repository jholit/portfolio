Personal Product Project

TennaJam — Dedicated Game-Server Hosting

TennaJam is a systems-focused product design project exploring how dedicated game-server hosting could feel calmer, clearer, and more trustworthy if it were designed around flow, structure, and long-term experience integrity rather than surface-level complexity.

Approx. 8 minute read
01 — Context

Why This Project Exists

Dedicated game-server hosting has existed for decades, yet the experience remains surprisingly fragile. Configuration workflows are dense, interfaces are intimidating, and small mistakes often feel irreversible.

TennaJam began as a personal response to that gap. The project is grounded in one core question:

What would server hosting feel like if it reduced uncertainty instead of amplifying it?

02 — Vision

Designing for Confidence

TennaJam is not about removing advanced control. It is about structuring power so it is understandable, predictable, and safe to use.

  • Clear, repeatable flow structures
  • Strong separation between overview and detail
  • Guided setup with sensible defaults
  • Consistent language for states and actions

The goal is a hosting environment that feels stable and calm, even when the underlying system is complex.

03 — Role

My Responsibility

TennaJam is a personal product project where my role is to ensure that the system behaves as intended from an experience and structural standpoint.

  • Defining product intent and guiding principles
  • Designing system flows and state behavior
  • Shaping the information architecture
  • Exploring interaction patterns through low-fidelity wireframes
  • Documenting design decisions and open questions

Business modeling, pricing strategy, and infrastructure implementation are intentionally out of scope.

04 — Structure

Core Interaction Model

TennaJam is intentionally organized around a small number of focused flows rather than a sprawling interface.

Day-to-day management

  • Home → Server Management → Server Details → Action

Hosting a new server

  • Select name, game, and region
  • Review recommended defaults
  • Enable advanced settings if needed
  • Confirm and launch
05 — Surfaces

Primary Screens

  • Home: Calm overview and entry points
  • Server Management: Server list with visible states
  • Server Details: Single source of truth for status and actions
  • Hosting Flow: Guided configuration experience
06 — Principles

What Guides the Design

  • Clarity over density
  • Structure before polish
  • Progressive control
  • Predictability builds confidence
07 — Reflection

What This Work Reinforced

This project reinforced a core belief: complex systems do not need to feel complex. When structure, language, and flow align with how people think, even infrastructure-heavy tools can feel approachable and safe.

08 — Status

Current State

TennaJam is an ongoing systems-design exploration. The work is intentionally early and conceptual, focused on correctness, clarity, and long-term coherence rather than execution.
