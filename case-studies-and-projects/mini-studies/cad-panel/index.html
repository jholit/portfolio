Case Study

Simplifying a CAD Parameter Panel

A compact UX exploration into reorganizing a dense parameter interface commonly found in CAD and engineering software. The focus is on clarity, organization, and cognitive load reduction for both beginners and expert users.

Approx. 4 min read
01 — Overview

1. Project Summary

This project examines how to simplify a cluttered parameter sidebar inspired by CAD tools. These interfaces often overwhelm users with long, unstructured lists of controls, lacking logical grouping or visual hierarchy.

The goal was to transform this environment into a structured, predictable interface that feels approachable while retaining technical precision.

  • Introduce hierarchy and logical grouping.
  • Reduce cognitive load through collapsible sections.
  • Support experts without overwhelming beginners.
  • Clarify labels and reduce ambiguity.
02 — Problem Space

2. The Problem

Technical UIs struggle when structure fails. This CAD-like panel showed four major issues:

  • No meaningful hierarchy across parameters.
  • Controls scattered without functional grouping.
  • High cognitive load for novices.
  • Experts slowed by unnecessary clutter.

The challenge: create a cleaner, more efficient way to browse and edit parameters without removing technical capabilities.

03 — Role & Scope

3. My Role & Responsibilities

This was a solo UX exercise focused on information architecture, flow clarity, and reducing cognitive load while respecting technical workflows.

Responsibilities

  • Analyze existing UI issues.
  • Identify natural parameter categories.
  • Design collapsible hierarchy.
  • Rewrite labels for clarity.
  • Explore alternate navigation structures.

This project focuses strictly on structure and usability — not visual polish or brand identity.

04 — Approach

4. Design Approach

The redesign followed a structured breakdown of the interface's issues and rebuilt the panel around predictable, task-based groupings.

Key Structural Improvements

  • Functional Grouping: Geometry, Dimensions, Materials, Constraints, Display.
  • Collapsible Sections: Reduce clutter while supporting advanced navigation.
  • Hierarchy & Rhythm: Improved spacing and labeling for predictable scanning.
  • Sidebar Concept: An optional left-navigation variant for expert users.
  • Plain-Language Labels: Reduce engineering jargon while staying precise.
05 — Solution

5. Final Solution

The redesigned CAD parameter panel features structured clusters, collapsible content, cleaner labels, and optional sidebar navigation. This reduces overwhelm and strengthens workflows for both new and experienced users.

06 — Outcomes

6. Key Outcomes

  • ~50% faster scanning for common parameters.
  • Lower cognitive load for beginners.
  • Experts can collapse irrelevant groups for focus.
  • Improved clarity and consistency across controls.
07 — Importance

7. Why This Matters

Engineering-focused companies benefit from designers who can simplify dense, high-information interfaces without removing the power users depend on. This project demonstrates the ability to bring clarity to complexity — an essential skill for CAD, simulation, and modeling environments.

08 — Learnings

8. Key Learnings

Hierarchy, grouping, and collapsible structure dramatically improve usability in technical tools. Clean information architecture reduces the need for visual polish and allows users to think more clearly while navigating complex systems.
